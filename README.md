# Graphanta v0.1.0-alpha.4

数学的な思考や表現を支援する、図・数式・アレー図・線分図作成アプリの初期動作版です。

## すぐに起動する

ZIPを展開して、フォルダー直下の **`index.html`** または **`Graphanta.html`** をEdgeかChromeで開いてください。

- WebサーバーやNode.jsは不要です。
- ファイルをダブルクリックして起動できます。
- `app.html` は開発用の入口なので、直接開かないでください。
- ファイル直開き時にブラウザ側で端末内の自動保存が禁止されている場合でも、作図・プロジェクトファイルの保存と読込は利用できます。

## 開発する

Node.jsを導入した環境で実行します。

```bash
npm ci
npm run dev
```

開発画面は `/app.html` で開きます。本番用ビルドは次の通りです。

```bash
npm run build
```

ビルド後には次を生成します。

- `dist/index.html`：GitHub PagesなどのWeb公開用
- `dist/Graphanta.html`：単体起動用
- ルートの`index.html`、`Graphanta.html`：単体起動用の最新版

## 主な機能

- SVGプロットエリア
- 選択、スクロール、クリック／範囲ズーム、複製、削除
- フリーハンド、線分、矢印、四角形、円・だ円、多角形
- 文字、数式
- アレー図、数直線・テープ図・線分図
- 白紙・方眼・座標表示、座標目盛り・数値・精度、吸着
- 変数フェーダーとアレー図・線分図の連動
- 元に戻す、やり直す
- プロジェクトと環境設定のJSON保存・読込
- PNGスクリーンショットの新規ウィンドウ表示
- 全画面プレゼンモード
- Web公開時のオフラインキャッシュ

関数グラフはアイコンとUIのみを設置し、v3で実装する予定です。


## alpha.4での追加・変更

- ツール名を「選択」「スクロール」「目盛り」へ整理
- 選択・スクロール・ズーム、線分・矢印・フリーハンド、四角形・円、文字・数式を展開式グループへ統合
- クリック中心ズームと、ドラッグ範囲を見切れない倍率へ合わせる範囲ズームを追加
- 白紙・方眼・座標の表示切替を追加し、方眼と座標の同時表示に対応
- 座標軸の目盛り、数値、精度、吸着を追加。原点0は常時表示
- 描画前に各ツールの線色・太さ・不透明度などを設定できるよう変更
- 線分・矢印の始点／終点座標入力、矢じりサイズ、描画後のラジアン回転を追加
- 円・だ円の中心／半径入力、長半径・短半径・離心率、回転を追加
- 目盛りツールを数直線・テープ図・線分図の3形式へ拡張
- テープ図・線分図に最大値と内分点フェーダーを追加。合計超過時は他の内分点を比例調整
- アレー図の行列数とシンボルを描画前に設定可能
- 数値・数式パラメータ入力を半角英数へ正規化

## alpha.3での修正

- 単体HTMLでスタイルシートが適用されず、標準ブラウザ表示になっていた問題を修正
- 通常のワイド画面では、プロット・縦型ツールバー・ツウィーク・fウィンドウを一画面に固定配置
- メニューバーを上部横一列へ再構成し、表示倍率・プロジェクト名・アプリ名を統合
- 文字の「G」を廃止し、座標軸と曲線を表すアプリアイコンへ変更
- ツウィークとfウィンドウをほぼ同じ高さにし、折りたたみ時は各パネルの幅も縮小
- 両パネルを折りたたんだ場合、空いた幅をプロットエリアへ還元
- 縦長画面ではメニューを2段化し、ツウィークとfウィンドウを画面下部の左右へ配置
- 選択枠にSVG属性を明示し、選択中のオブジェクトが黒く塗りつぶされる問題を修正
- 1366×768、1920×1080、768×1024でページ全体のスクロールが生じないことを確認

## alpha.2での修正

- HTMLをダブルクリックした際に真っ白になる問題を修正
- JavaScriptとCSSを内包した単体HTMLを追加
- `file://`環境ではService Workerを登録しないよう修正
- localStorageやIndexedDBが拒否されても起動を続けるよう修正
- UUID生成が使えない環境向けの代替処理を追加
- 起動時エラーが発生した場合に、白画面ではなく内容を表示するよう修正
